The `git add` command is one of the most fundamental Git commands. It **stages changes** from your working directory, preparing them for the next commit. Think of it as telling Git "these are the changes I want to include in my next snapshot."

**Commands to stage related changes:**
```bash
# Stage navbar and styling changes together
git add index.html style.css script.js

# Create commit for navbar feature
git commit -m "Add responsive navigation bar"

# Stage typo fixes separately
git add about.html

# Create separate commit for content fixes
git commit -m "Fix typos in about page"
```

## Common Workflows and Best Practices

### 1. Check Before You Stage
```bash
git status          # See what files are changed
git diff           # See what changes you made
git add <files>    # Stage specific files
git diff --staged  # Review staged changes
```

## Troubleshooting Common Issues

### Unstaging Files (Oops, staged wrong file!)
```bash
git reset HEAD <filename>    # Unstage specific file
git reset HEAD .            # Unstage all files
```

<Mermaid 
  chart={`
    graph LR
      Staged[‚úÖ Staged File] -->|git reset HEAD file| Working[üìù Working Directory]
      
      style Staged fill:#c8e6c9
      style Working fill:#fff3e0
  `}
/>

### Checking What's Staged
```bash
git status           # Overview of staged/unstaged files
git diff --staged   # See exact changes that are staged
git diff --cached   # Same as --staged
```
