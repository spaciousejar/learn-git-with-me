## What is Git?

Git is a **distributed version control system** that tracks changes in computer files and coordinates work on those files among multiple people. It was created by Linus Torvalds in 2005 for development of the Linux kernel.

## Git's Three States

Git has three main states that your files can reside in:

<GitWorkflowDiagram type="basic" />

1. **Working Directory**: Where you modify files
2. **Staging Area**: Where you prepare changes for commit
3. **Repository**: Where Git permanently stores snapshots

## How Git Thinks About Data

Unlike other VCS that store differences, Git thinks of data as snapshots.

## Basic Git Workflow

Here's what a typical Git workflow looks like:

<Mermaid 
  chart={`
    graph TD
      Start([Start Project]) --> Init[git init]

      Decision -->|No| Push[git push]
      Push --> End([Project Updated])
