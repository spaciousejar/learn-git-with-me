# Labelling Issues / PRs

On GitHub, labels are a way to categorize issues and pull requests (PRs) by topic, priority, or other criteria. Some common labels used are:

- `Bug`
- `Duplicate`
- `Enhancement`
- `Feature request`
- `High priority`
- `Needs feedback`
